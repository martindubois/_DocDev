[ ] S'assurer que l'ordinateur a été préparé tel que décrit par
    «Procedures/HCK/PreparerOrdinateurTest.txt»;
[ ] Installer les logiciels à tester sur l'ordinateur de test
    - Si nécessaire, redémarrer l'ordinateur de test.
[ ] Outils - HCK Studio - Sur le serveur WHCK
    - Si l'ordinateur (ou une de ces autres personnalités) de test est déjà
      dans la liste des ordinateurs connus, la retirer.
[ ] Désactiver le pare-feu Windows pour les réseaux privés;
   
Note   Si l'ordinateur est configuré pour une certification de système, il
       ne faut pas désactiver le pare-feu, il faut alors ajouter une règle
       entrante et une règle sortante pour permettre les communications sur le
       sous-réseau local.

[ ] Exécuter \\WHCK\HCKInstall\Client\Setup.exe;
[ ] Outil - HCK Studio - Sur le serveur WHCK
    - Attendre que l'ordinateur apparaisse dans le «Default Pool»;
    - Transferer l'ordinateur dans le pool approprié;
    - Changer son état pour «Ready»;
    - Attendre que la phase d'initialisation soit terminée et que l'état de
      l'ordinateur soit vraiment «Ready».
