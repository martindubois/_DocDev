[ ]	S'assurer que l'ordinateur a été préparez tel que décrit par
	"Procedures/HCK/PreparerOrdinateurTest.txt";
[ ]	Installer les logiciels a tester sur l'ordinateur de test
	- Si necessaire, redémarrer l'odrinateur de test.
[ ]	Configurer l'ordinateur de test pour utiliser l'adresse IPv4 statique qui lui
	est attribuee;
[ ]	Lancer le "HCK Studio" sur Li pour vous assurer que le serveur de base de
	données fonctionne bien
	- Si l'ordinateur de test est deja dans la liste des ordinateurs connue, la
	  retirer.
[ ]	Exécuter "\\Li\HCKInstall\Client\Setup.exe";
[ ]	Configurer l'adresse IP statique associé à cette ordinateur;
[ ]	Vérifier que le pare-feu Windows est désactivé pour les réseaux privés;
	Note :	Si l'ordinateur est configurer pour une certification de systeme, il
		ne faut pas desactiver le pare-feu, il faut alors ajouter une regle
		entrante et une regle sortante pour permettre les communication sur
		le sous reseau locale.
[ ]	Dans "HCK Studio"
	- Attendre que l'ordinateur apparaisse dans le "Default Pool";
	- Transferer l'ortinateur dans le pool approprié;
	- Changer son état pour "Ready";
	- Attendre que la phase d'initialisation soit terminé et que l'état de
	  l'ordinateur soit vraiment "Ready".
